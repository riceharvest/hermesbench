# HermesBench Long-Run Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task. Treat every code-producing task as TDD: write the failing test, run it, implement, rerun targeted tests, then rerun the relevant full gate.

**Goal:** Turn HermesBench from a credible public scaffold into a durable, contamination-resistant benchmark program with official private/fresh/anchor runs, a real submission API, a production leaderboard, and repeatable governance.

**Architecture:** Keep the benchmark runner, task corpus, API, and website loosely coupled. The Python package remains the canonical task/schema/scoring implementation; the API ingests normalized run JSON; the website reads leaderboard/demo/API data; official runs are generated by a maintainer-only runner using private task packs and archived scoring evidence.

**Tech Stack:** Python 3.11+, uv, pytest, YAML/Markdown task specs, JSONL result/submission storage initially, optional Hono/FastAPI/Cloudflare Worker API later, static/Next.js leaderboard, GitHub Actions CI, GitHub Releases for benchmark versions.

Example test shape:

```python
def test_public_dev_tasks_are_not_marker_only():
    tasks = discover_tasks('public-dev')
    shallow = []
    for task in tasks:
        prompt = task.prompt.lower()
        rubric = task.scoring_rubric.lower()
        marker_only = 'task_id:' in task.prompt and 'verified: true' in task.prompt and len(task.deterministic_checks) <= 3

            shallow.append(task.metadata['id'])
    assert not shallow
```

## Risk Register

| Risk | Mitigation |
|---|---|
| Public tasks remain too shallow | Add task-quality tests and convert by category waves |
| Private holdout leaked | Keep real private pack outside public repo; public repo only has templates |
| CI too slow from legacy ML deps | Split legacy dependencies into optional extra |
| API duplicates Python schema logic | Wrap `src/hermesbench/api.py` and `schemas.py`; do not rewrite validation elsewhere |
| Website becomes fake marketing page | Require demo data, result detail pages, and score evidence rendering |
| Agents game public/dev tasks | Treat public/dev as development only; official scoring uses private/fresh packs |
| LLM judge unreliability | Use judge only as secondary score where objective scoring is impossible |
| False-done not captured | Harden transcript/evidence checks and include false-done rate in leaderboard |
| Old qwen-mtp files confuse users | Add process/provenance docs now; later move to legacy/archive branch |

## Definition of “Officially Launched”

HermesBench can claim official leaderboard integrity only when all of these are true:

1. Public/dev tasks are realistic enough to be useful for local development.
2. Real private holdout tasks are not in the public repo.
3. Fresh rolling wave process has produced at least one current wave.
4. Anchor set is frozen and versioned.
5. Official run policy is published.
6. Result submissions API is deployed or the manual submission process is documented and enforced.
7. Website shows actual leaderboard/detail data, not only placeholders.
8. CI passes on every release commit.
9. At least one official run archive exists with manifest, result hash, and score summary.
10. README clearly distinguishes unofficial self-submitted public/dev runs from official private/fresh/anchor leaderboard runs.
