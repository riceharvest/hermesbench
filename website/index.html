Methodology

Scores come from normalized result JSON and deterministic/test evidence. Official scores require maintainer-controlled private/fresh/anchor packs and archived manifests. LLM judges are secondary only.

Tasks

Task pages are static for v0.1. Each task records category, fixtures, expected artifacts, scoring checks, contamination notes, freshness window where relevant, and hidden-check policy.

Submissions

For v0.1, self-submitted public/dev results are unofficial. Run uv run hermesbench run --agent mock --suite public-dev, score locally, inspect for secrets/local paths, then publish normalized JSON.

Why real-world agent benchmarks are hard

Agents can overfit static prompts, claim success without artifacts, or pass shallow checks while missing operational details. HermesBench separates development tasks from hidden/private/fresh/anchor evaluation and renders evidence so users can audit why a score was awarded.
